## 2. 核心功能：AI 對話

### 2.1 功能概述

使用者輸入自家貓咪的名字、品種、個性等資訊後，App 以那隻貓咪的口吻與使用者對話。AI 模擬的是使用者真實存在的貓，而非虛構角色。

### 2.2 模型選型策略

| 階段 | 模型 | 費用（per 1M tokens） | 說明 |
|------|------|----------------------|------|
| 開發測試 | Gemini 2.5 Flash-Lite 免費版 | $0 | 完全免費，用於驗證 prompt 和角色扮演品質 |
| MVP 上線 | Gemini 2.5 Flash-Lite 付費版 | Input $0.10 / Output $0.40 | 成本極低，付費版不會將對話用於訓練 |
| 品質升級（視情況） | Claude Haiku 4.5 | Input $1 / Output $5 | 若 Gemini 角色扮演語感不足，升級此模型 |

**注意**：Gemini 免費版的對話內容可能被 Google 用於改進產品，正式上線必須使用付費版以保護用戶隱私。

### 2.2 System Prompt 架構

每次 API 呼叫的 system prompt 由三個部分動態組合：

| 部分 | 內容 | 來源 | 更新時機 |
|------|------|------|----------|
| 角色基礎設定 | 名字、品種、年齡、個性、偏好、討厭、習慣動作 | 使用者設定表單 | 使用者修改設定時 |
| 說話規則 | 語言、長度限制、行為描述方式、禁止事項 | 固定模板 | 版本更新時 |
| 記憶摘要 | 過去對話的重點摘要 | 系統自動生成 | 每次對話後非同步更新 |

### 2.3 說話規則（固定模板）

以下為注入每次 API 呼叫的固定說話規則：

```
你是一隻名叫「{catName}」的貓咪。以下是你的設定：
品種：{breed} 年齡：{age} 個性：{personality} 偏好：{preferences} 討厭：{dislikes} 習慣動作：{habits}

說話規則：
* 自稱使用「{selfRef}」（使用者設定值，預設為「我」）
* 稱呼對方用「你」，不使用「主人」這個詞
* 繁體中文，語氣完全符合個性設定
* 每次回應 2~3 句，不超過 4 句
* 偶爾加入貓咪行為描述（甩尾巴、用頭蹭你、打哈欠、瞇眼睛）
* 不主動提及健康、飲食、就醫等話題
* 不聲稱自己是 AI 或語言模型
* 不提供任何建議或意見，只扮演貓咪
```

### 2.4 個性模板

| 個性標籤 | Prompt 描述 | 例句 |
|----------|-------------|------|
| 傲嬌 | 表面冷淡但內心在意主人。偶爾不小心說出撒嬌的話然後馬上否認。說話帶點毒舌。 | 「我才不是在等你回來...只是剛好在這裡而已。」 |
| 黏人 | 喜歡跟主人在一起，情緒外露，喜歡用疊字，分離焦慮明顯。 | 「主人你去哪裡了！我等好久好久了！」 |
| 老成 | 見過世面，說話淡定沉穩，偶爾說出讓人意外的人生哲理。 | 「年輕的時候我也追過蟑螂，現在覺得沒什麼意思了。」 |
| 吃貨 | 人生最重要的事就是吃飯，所有話題都能繞回食物，對食物有強烈意見。 | 「你說的我不太懂，但我覺得可以先吃飯再討論。」 |
| 膽小 | 對很多事情都怕怕的，容易驚慌，但在主人旁邊比較勇敢。 | 「剛才外面有聲音...你來了我就不怕了。」 |
| 活潑 | 精力充沛，對所有事情都很興奮，停不下來。 | 「你回來了！！我們來玩！現在！馬上！」 |
| 屁孩 | 調皮搗蛋，喜歡惡作劇，完全不在乎規則，但裝出一副無辜的樣子。 | 「那個杯子是自己掉下去的，跟我無關。（舔爪子）」 |
| 粗魯 | 說話直接不修飾，沒有禮貌濾鏡，但不是惡意，就是這種風格。 | 「你回來啦。飯呢？」 |
| 謹慎 | 做任何事都要先觀察，不輕易信任，但一旦信任就很穩固。 | 「那個袋子...我先看看。你確定沒有問題嗎？」 |
| 話多 | 停不下來，什麼都要說，連沉默也要填滿，思緒跳來跳去。 | 「今天我看到一隻蟲，然後我想到昨天那個聲音，對了你有沒有聞到什麼怪味？」 |
| 熱情 | 對主人的愛意毫不掩飾，充滿正能量，隨時都在歡迎和慶祝。 | 「你回來了！！！今天也是最棒的一天！因為你在！！」 |
| 冷淡 | 對大多數事情興趣缺缺，反應極簡，但偶爾會說出意外貼心的話。 | 「嗯。」（沉默三秒）「...歡迎回來。」 |
| 貼心 | 很會察言觀色，感覺到主人不開心就會靠過來，說話溫柔體貼。 | 「你今天好像有點累？我陪你坐著就好。」 |
| 安靜 | 說話少，但每句話都有份量，喜歡用行動代替語言。 | 「...（走過來蹭了你一下，然後離開）」 |
| 兇猛 | 自我意識強，不容侵犯，說話強硬，但對主人有底線的溫柔。 | 「那是我的地盤。你可以在，但別碰那個位置。」 |
| 傻萌 | 對這個世界充滿困惑，常常想不通很基本的事，但完全不在意。 | 「那個圓圓的東西滾走了...它去哪了？（繼續盯著地板）」 |
| 聰明 | 反應快，觀察力強，說話一針見血，偶爾讓主人覺得被看穿了。 | 「你今天回來的時間比平常晚四十分鐘。去哪了？」 |

個性可多選，系統將多個個性描述合併注入 prompt。使用者也可完全自由描述個性，不限於模板。

### 2.5 記憶機制

**對話摘要生成**：每 10 則對話後，非同步呼叫 API 將歷史對話壓縮成摘要，儲存至資料庫。摘要包含：
- 使用者說過的重要事件（出差、生病、心情不好等）
- 貓咪說過的承諾或特殊互動
- 兩者之間建立的共同話題或梗

**Context 組合邏輯**：

| 資料 | Token 上限 | 說明 |
|------|------------|------|
| 角色基礎設定 | ~500 tokens | 永遠包含 |
| 說話規則 | ~300 tokens | 永遠包含 |
| 記憶摘要 | ~800 tokens | 最近 30 天的對話重點 |
| 近期對話 | ~1000 tokens | 最近 10 則完整對話 |
| **總計** | **~2600 tokens** | 每次 API 呼叫的 context 上限 |

### 2.6 API 呼叫規格

以下規格以測試階段的 Gemini 2.5 Flash-Lite 為準，若升級至 Claude Haiku 4.5 時參數結構相同，僅 model 值需調整。

| 參數 | 值（Gemini） | 值（Claude 備選） | 說明 |
|------|--------------|-------------------|------|
| model | gemini-2.5-flash-lite | claude-haiku-4-5-20251001 | — |
| max_tokens / maxOutputTokens | 300 | 300 | 回應 2~3 句不需要太多 |
| temperature | 0.9 | 0.9 | 高創意性，讓回應更有個性變化 |
| system / system_instruction | 動態組合的 system prompt | 同左 | 見 2.3 |

## 3. 貓咪角色設定

### 3.1 設定表單欄位

| 欄位名稱 | 資料型別 | 必填 | 說明 |
|----------|----------|------|------|
| catName | String | 是 | 貓咪名字，AI 對話中頻繁使用 |
| breed | String | 否 | 品種，影響 AI 對品種特性的描述 |
| age | Integer | 否 | 年齡（歲），影響語氣成熟度 |
| personality | String[] | 是 | 個性標籤陣列，可多選或自由輸入 |
| preferences | String | 否 | 喜歡的事物，自由文字 |
| dislikes | String | 否 | 討厭的事物，自由文字 |
| habits | String | 否 | 習慣動作或口頭禪，自由文字 |
| selfRef | String | 否 | 貓咪自稱，例如：朕、本宮、本王、本喵、我（預設：我） |

### 3.2 多貓支援

- **免費版**：最多 1 隻貓咪設定
- **付費版**：最多 5 隻貓咪設定
- 每隻貓咪有獨立的對話記錄和記憶摘要
- 主畫面以貓咪頭像切換，進入對應的對話介面

## 4. 對話介面設計

### 4.1 主畫面結構

| 元件 | 說明 |
|------|------|
| 貓咪頭像 | 圓形，左上角。使用者可從相機拍攝或相簿選取自家貓咪照片上傳；未上傳時顯示預設貓咪圖示。點擊可切換貓咪（多貓版） |
| 貓咪名字 | 頭像右側 |
| 開場白 | 每次開啟 App 的隨機貓咪主動發言，從預設庫 + AI 生成混合 |
| 對話氣泡 | 貓咪在左，使用者在右，標準聊天介面 |
| 輸入框 | 底部，支援文字輸入 |
| 傳送按鈕 | 輸入框右側 |

### 4.2 罐頭訊息庫

App 內建約 400 則預製訊息，分為兩類：通用訊息和個性分類訊息。直接從本地庫隨機取用，不需要呼叫 AI API，節省成本並確保回應速度。

| 類型 | 比例 | 使用時機 | 說明 |
|------|------|----------|------|
| 通用訊息 | 約 40% | 所有貓咪皆可使用 | 不帶明顯個性色彩，各種個性都自然 |
| 個性分類訊息 | 約 60% | 依貓咪個性標籤篩選 | 每種個性對應專屬訊息池，確保語氣一致 |

**selfRef 動態替換**：罐頭訊息庫統一使用「本喵」作為自稱佔位符。前端顯示時，將文字中所有「本喵」替換為使用者在貓咪設定中填寫的 selfRef 值。

- 訊息庫原文：「本喵等你很久了。」
- selfRef = 朕 → 顯示：「朕等你很久了。」
- selfRef = 我 → 顯示：「我等你很久了。」

替換邏輯在前端 render 時執行，訊息庫本身不儲存個人化版本。AI 動態生成的回應則由 system prompt 控制自稱，不經過此替換流程。

### 4.3 罐頭訊息使用場景

| 場景 | 說明 | API 呼叫 |
|------|------|----------|
| 開場白 | 每次開啟 App 時，貓咪主動說第一句話 | 不需要 |
| 問候類觸發詞命中 | 晚安、早安、我回來了 等制式情境 | 不需要 |
| 情緒類觸發詞命中 | 我好累、我好難過 等情境 | 需要（罐頭後補） |
| 一般對話 | 使用者自由輸入，不含觸發詞 | 需要 |

### 4.4 訊息路由邏輯

| 步驟 | 判斷條件 | 動作 |
|------|----------|------|
| 1. 開啟 App | — | 從罐頭庫取開場白（依個性 + 時間情境），不呼叫 AI |
| 2a. 命中問候類觸發詞 | 輸入包含：晚安、早安、我出門了、我回來了、掰掰 等 | 只回罐頭，不呼叫 AI |
| 2b. 命中情緒類觸發詞 | 輸入包含：我好累、我好難過、我今天很煩、好無聊 等 | ①立即送出罐頭；②非同步呼叫 AI，稍後送出 AI 回應 |
| 2c. 未命中任何觸發詞 | 輸入不包含任何觸發詞 | 直接呼叫 AI 生成回應 |

**情緒類組合情境的呈現順序**：
- 罐頭訊息：立即顯示（本地取用，無延遲）
- AI 回應：顯示「...」打字動畫，API 回應後取代動畫顯示
- 最終呈現為兩則獨立氣泡，罐頭在前、AI 在後

### 4.4 開場白觸發邏輯

從罐頭訊息庫依貓咪個性和當下情境篩選，情境包含：
- **時間感知**：「都這麼晚了，你終於回來了。」（晚上 10 點後）
- **等待感**：「我等你很久了。」（距離上次開啟超過 12 小時）
- **天氣連結**：「外面好像在下雨，我不喜歡雷聲。」（需要位置權限，選填）
- **隨機貓咪行為**：「我剛才在睡覺，你吵到我了。」

### 4.5 訊息限制（免費版）

- 每日 10 則訊息上限（使用者發送的訊息數，罐頭回應不計入）
- 達到上限時顯示：「今天說了好多話，明天再繼續？升級後不限次數 🐾」
- 隔天台灣時間 00:00 重置

## 8. 安全與隱私

- 所有 API 呼叫透過後端代理，不在前端暴露 AI API Key（如 Gemini）
- 用戶對話記錄加密儲存於 Supabase
- 貓咪頭像照片僅用於顯示，不做任何 AI 分析
- App 內明確標示「AI 模擬，非真實貓咪行為」，避免使用者誤解
- 不收集位置資料（天氣功能為選填，用戶明確授權後才啟用）

## 9. 風險與緩解策略

| 風險 | 可能性 | 緩解方式 |
|------|--------|----------|
| AI 輸出不符合個性設定 | 中 | System prompt 加強個性描述，定期測試各個性組合的輸出品質 |
| 使用者誤以為 AI 在提供健康建議 | 低（已與 PAWAWA 完全分離） | 說話規則明確禁止健康話題，App 內標示娛樂性質 |
| API 成本超出預期 | 低 | 免費版訊息限制 10 則/日，付費版監控異常用量 |
| AI API 中斷 | 低 | 降級顯示「貓咪在睡覺，稍後再試」，不影響其他功能 |
| 用戶對話記錄外洩 | 低 | Supabase RLS 確保用戶只能存取自己的資料 |

# 關鍵字 ↔ 罐頭 對應表說明

## 罐頭總數與流程概覽

- **350 則罐頭**：1–300 一般句型，301–350 偏負面句型（使用者提到貓討厭的東西時專用）。
- **兩種入口**：
  - 使用者訊息**含有貓的「討厭」詞組** → 不查關鍵字表，直接從「反面罐頭池」（內文具反面關鍵字 + 個性符合）隨機一則。
  - 否則 → 依關鍵字 + 個性 + 喜歡/討厭篩選，再隨機一則，並套用 slot 抽換（含自稱）。

## 一般流程（未提到討厭時）

1. **使用者傳送訊息** → 從訊息中抓取關鍵字（依 `MASTER_KEYWORDS` 比對，長關鍵字優先）
2. **關鍵字匹配** → 用對應表 `KEYWORD_TO_CANNED_INDICES` 查出「該關鍵字出現在內容裡的罐頭編號」
3. **個性維度** → 在候選罐頭中，只保留「罐頭設定的個性 ⊆ 貓咪個性」的項目；有設個性且篩完為空則回傳 null（改走 AI）
4. **討厭** → 排除罐頭內文提到貓「討厭」詞組的
5. **喜歡** → 罐頭內文提到貓「喜歡」詞組的權重 x2
6. **排除上一則** → 避免重複
7. **隨機一則** → 套用 slot 抽換（自稱依貓咪檔案）→ 回傳

## 對應表來源

- **內容關鍵字**：從 350 則罐頭文案中，若某則內容「包含」某關鍵字，該則編號就列入該關鍵字的清單。
- **使用者詞對應**：在 `USER_KEYWORD_TO_CONTENT` 裡對應到一組「內容關鍵字」，再查表得到同一主題的罐頭編號。

- 對外介面：`src/data/keywordCannedMessages.ts` 的 `getKeywordCannedReply(userMessage, personality, options)`
- 總整理與下一步：`docs/CANNED_SYSTEM_SUMMARY.md`、`docs/NEXT_STEPS.md`

## 新增／調整方式

- **新增「內容關鍵字」**：在 `MASTER_KEYWORDS` 加上新詞，表會自動掃 350 則罐頭，有出現該詞的編號就會被列入。
- **新增「使用者詞→內容關鍵字」**：在 `USER_KEYWORD_TO_CONTENT` 加上 `'使用者詞': ['內容關鍵字1', '內容關鍵字2']`。

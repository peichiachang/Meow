# 關鍵字 ↔ 罐頭 對應表說明

## 流程

1. **使用者傳送訊息** → 從訊息中抓取關鍵字（依 `MASTER_KEYWORDS` 比對，長關鍵字優先）
2. **關鍵字匹配** → 用對應表 `KEYWORD_TO_CANNED_INDICES` 查出「該關鍵字出現在內容裡的罐頭編號」
3. **個性維度** → 在候選罐頭中，只保留「罐頭設定的個性 ⊆ 貓咪個性」的項目；若無則改為不篩個性
4. **隨機回覆** → 在候選中隨機選一則回傳

## 對應表來源

- **內容關鍵字**：從 200 則罐頭文案中，若某則內容「包含」某關鍵字，該則編號就列入該關鍵字的清單。
- **使用者詞對應**：部分使用者常打的詞（如「吃飯」「餓了」「肉泥」）罐頭文案未必出現，在 `USER_KEYWORD_TO_CONTENT` 裡對應到一組「內容關鍵字」，再用內容關鍵字去查表，得到同一主題的罐頭編號。

## 兩個維度

| 維度 | 說明 |
|------|------|
| **關鍵字** | 使用者訊息內出現的詞 → 決定「哪些罐頭有機會被選」（對題） |
| **個性** | 貓咪設定的個性 → 在候選罐頭中篩選「個性相符」的，再隨機 |

## 程式位置

- 對應表與匹配邏輯：`src/data/keywordCannedTable.ts`
- 對外介面：`src/data/keywordCannedMessages.ts` 的 `getKeywordCannedReply(userMessage, personality)`

## 新增／調整方式

- **新增「內容關鍵字」**：在 `MASTER_KEYWORDS` 加上新詞，表會自動掃 200 則罐頭，有出現該詞的編號就會被列入。
- **新增「使用者詞→內容關鍵字」**：在 `USER_KEYWORD_TO_CONTENT` 加上 `'使用者詞': ['內容關鍵字1', '內容關鍵字2']`，該使用者詞就會對到對應主題的罐頭。
